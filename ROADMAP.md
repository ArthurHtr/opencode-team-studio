# Roadmap

This roadmap outlines the planned direction for OpenCode Team Studio. It is not a commitment and may change based on community feedback, resource availability, and OpenCode platform evolution.

## Alpha

- [x] Docker-based local deployment
- [x] Visual management of OpenCode agents and relationships
- [x] Skills, MCP, model, and permission configuration
- [x] Transactional configuration backups and restoration
- [x] Configuration preservation (comments, unknown fields)
- [x] Public documentation
- [x] Linux deployment

## Beta

- [ ] macOS and Windows/WSL2 documented and tested
- [ ] Configuration diagnostics (validation warnings, deprecation notices)
- [ ] Better display of inherited values
- [ ] Provider/model option management
- [ ] Team import/export (shareable configurations)
- [ ] Enhanced integration tests
- [ ] Performance optimization for large teams (50+ agents)

## Stable (1.0.0)

- [ ] Versioned OpenCode compatibility policy
- [ ] Migration guides for config format changes
- [ ] Comprehensive test coverage
- [ ] Security audit
- [ ] Complete documentation
- [ ] Community contribution workflow established

## Later

- [ ] Shareable team templates
- [ ] Graph analysis (circular dependencies, orphaned agents)
- [ ] Permission excess detection
- [ ] Plugin or extension system
- [ ] Multi-user collaboration (planned, not confirmed)

---

This roadmap is subject to change. Check [releases](https://github.com/ArthurHtr/opencode-team-studio/releases) for the latest updates.
